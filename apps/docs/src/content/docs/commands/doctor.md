Validate your configuration and project setup.

## Usage

```bash
omnidev doctor
```

## What it checks

- Configuration file validity (`omni.toml`)
- Capability source availability
- Provider configuration
- Required dependencies
- Common setup issues

Run `omnidev doctor` when:

- Something isn't working as expected
- After initial setup to verify everything is correct
- Before reporting issues
